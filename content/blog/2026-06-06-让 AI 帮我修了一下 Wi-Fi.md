自从去年搬到现在的出租屋，我一直都连着自己的路由器，今天突发奇想，让 Codex 帮我检查网络，没想到还真有收获。

我是在合租房，客厅有一个光猫和公共的路由器，室友们基本都是连这个。但是我刚搬来那会儿，主要用 Windows 台式电脑，没有无线网卡，从客厅走线也比较麻烦，当时我灵机一动，把之前买的路由器做个桥接，即让它配置后连接客厅路由器，开启 Wi-Fi，然后用一根网线连上台式机，我的电脑也就能上网了。

但是最近半年，我几乎很少用 Windows 主机，而是用 MacBook 来做绝大多数事情。但我的习惯依然是连自备路由器的网络，这中间网络偶尔会比较慢，但我觉得不是不能接受。

刚好我电脑有 Codex，它既然能在本地大展拳脚，那么检查网络应该不在话下。于是我把情况说给 Codex，它检查一下我现在连的自备 Wi-Fi 后，说有 10% 丢包，不算很稳，让我切换到客厅 Wi-Fi。

不承想，我随手切换后，说客厅主路由 Wi-Fi 更稳，因为丢包为 0%，延迟更低。我有些惊讶，过去习以为常的习惯，原来也存在问题。

原因是我过去联网路线为：光纤 → 光猫 → 客厅路由器 → 自备路由器 → MacBook，一段数据会在 Wi-Fi 里多转一次，信号自然会打折扣。而我绕开自备路由器，连到客厅 Wi-Fi，就少一道流程，网络会更加流畅。

![Screenshot 2026-06-06 at 22.09.18.jpeg](https://img.liangmouyin.com/2026/06/1af05044fe31759d92d9d0ab5f70f5b2.jpeg)

Codex 给我的建议是主用客厅 Wi-Fi 就好，因为它更稳定，且延迟低，只有在客厅 Wi-Fi 信号差时，再考虑切换到自己的 Wi-Fi。不过经此一说，我甚至觉得没有必要再开自己的路由器，所以我直接拔掉插头，将屏幕挂灯和台灯都切换到客厅 Wi-Fi，现在就只有用 Windows 电脑时，我才有必要开路由器。

如果你觉得自己的网络有问题，也能按照我的思路来排查。哪怕不用 Codex，你用腾讯的 WorkBuddy 或者 Marvis 也能见效，毕竟这对 AI 来说只是基础活，够用就行。下面是个简单的提示词，仅供参考：

```
帮我排查网络。先不要猜结论。
请按这几步来：
1. 确认我现在连的是哪个 Wi-Fi、IP、默认网关。
2. ping 当前网关，判断本地 Wi-Fi/路由器这一跳是否丢包或延迟抖动。
3. ping 一个公网 IP，比如 1.1.1.1，判断外网是否稳定。
4. 如果我有多个 Wi-Fi，让我切换后用同样方法重测。
5. 最后用表格总结：哪个更快、哪个更稳、为什么。
```

最后 AI 会请求一些权限，你全部同意后，按照要求切换网络，你就能得到一份新的网络使用建议。

我过去总用 AI 来处理复杂的任务，如写代码、折腾博客、处理文档、设计网站，可全然没想过还能改善网络，处理隔三岔五就要忍受的小问题。

这些问题日常且微小，但频率之高，又不能忽视。好在现在有了 AI，除了让它处理重活，也能改善眼前的生活，它俨然成了电脑里的瑞士军刀。毕竟，杀鸡用牛刀还是菜刀其实并不讲究，顺手才是重点。

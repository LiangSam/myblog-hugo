周末我在家里吭哧瘪肚干活，突然冒出一个新想法，立刻让 Claude 大展拳脚。没想到我打开设置，查看 Usage 额度时，发现本周限额只剩 1%，既然如此，就不能像以往那样散漫式聊天，必须换个方式。

于是，我用半个多小时梳理需求，写清前因后果，规范好素材，再一起发给 Claude Code，模型选择 Opus 4.6。聊完一个对话后，Claude 进一步确认需求，我说我要的不是这个，它明白，说那我开始写。孰料写到一半，噩耗传来，我的额度已到上限，等 4 月 13 号晚上重置。

你要问我什么时候最紧张，就是额度已用 99%，但我还指望它帮我干活，所以迫切希望它先干完，质量姑且不论，我后面还能调整。可 Claude 活已经干到一半，我一时想不出还有谁能接手它的工作。

这感受像极了在外逛街，手机只剩 10% 的电，期望能挺到回家，可还没出发就已经没电，只能走路回去，打车是不可能的。

手机或许能借助共享充电宝充电，但和 Claude Code 切换到其他模型有本质差异，后者你不能保证国产大模型的 API 是否也能这么卖力。

最后我选择了 ChatGPT 的 Codex，把之前的对话如数给它。最后明显能感到 ChatGPT 做得很用心，可结果差强人意，略显保守，最后还是得我自己来弄，只能作为参考。

这其实涉及一个问题，就是什么工作应该外包给 AI。我认为对结果起主要作用的事情必须由自己深度介入，这并非是说我比 AI 强，而是我需要把关，凭借我的判断和经验来拍板。

ChatGPT 善守正，风格规整无纰漏，但风格偏向讨好，让人觉得太圆滑、废话多、不犀利；Claude 能出奇，很少阿谀奉承（3 月底泄露的代码就明确要求「不讨好用户」），废话少，风格独树一帜，干活娴熟。

最后好在把活干完了。

其实没有 Claude Code 之前，我自己也能做很多活，而有了它之后，我反倒是变懒了，想着有个工具能够做得不错，那何必自己花费精力。但我忽略了额度会有用完一天，甚至账号也有被封的一刻，抛去 AI，才发现非代码的创作也只能靠自己。

我并不主张祛魅 AI，因为现在 AI 可谓飞入寻常百姓家，我过年给家人推荐了豆包，让他们有事没事就问问，总比信短视频好。

周末听到 Hum 说，祛魅 AI 等于祛魅空调。这话隐含的立场是别纠结用 AI 好不好，把它当作工具，先操练起来。不过空调带来而凉爽是客观的，而 AI 会输出观点，并影响你对一件事的看法和判断，而且迭代极快——但几年后空调的用法依然不变。

最后我的经验是，下次用 AI 干活前少说废话，这样能节省不少额度，不然就是我在给它提供情绪价值了。

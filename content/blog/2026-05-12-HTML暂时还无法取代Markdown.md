不知道多少次，我在公众号刷到一堆自媒体说 Claude Code 的负责人说已经放弃 Markdown，全面拥抱 HTML，甚至连卡帕西（Karpathy）也在推特说它确实好。不过这些账号大部分都是英译汉，没有给出深入的观点。

但我非常困惑，Markdown 还未全面普及，现在又有新的浪潮来了？

就我所知，我身边的同事大抵是不碰 Markdown 的，基本都用 WPS 或 Microsoft Office，你问 Obsidian 是什么，只怕都是一脸茫然。但这不代表我要抬高 Markdown 并贬低传统办公软件，在我看来 Word、Markdown、HTML 都只是工具在不同需求下的截面罢了，没人会说锤子要取代螺丝刀。

Markdown 的好处在于它的易用性。只要掌握些许符号，就能排版出多级标题、粗体、斜体、无序列表、代码块、高亮等丰富样式，而且比 Word 好在不用操心行间距、字体、格式，加上主流笔记软件所见即所得的体验，确实是 HTML 无法给予的。

相比之下，HTML 对人类写作者来说更像是一个黑盒。虽然它能呈现出排版更华丽、甚至带有交互功能的页面，但其背后堆积如山的代码逻辑，对于非开发者来说几乎不可读。即便现在有 AI 代劳，可一旦你想微调一段文字的颜色或间距，还得在代码丛林里翻找，这种挫败感是致命的。

其实，这些人提倡 HTML，更多是站在 AI 输出载体的视角——既然 AI 能直接生成结构完美的富文本 UI，又何必局限在简陋的 Markdown 框架里？

但回到现实生活中，如果你要交付一份文件给领导、老师或同事，格式的可编辑性和通用性才是第一优先级。如果你给对方发一个 HTML 文件，对方可能连如何修改都无从下手。这种时候，Office 三件套地位依然尊贵。

不少人看到讨论就焦虑起来，其实选择工具本来是自由的。即便以 Obsidian、Notion 等笔记软件如火如荼，但我认识的依然有人在用印象笔记、Notion、Flomo，它们并不冲突，只要这些软件服务商没有倒闭，那就还能用到很久。

除了数字工具，我最近甚至开始使用纸质笔记本。我斥资买了几本 Field Notes（尺寸比 A6 小，我关注的一些作者也在用，遂购买之），体验下来，其实和淘宝那种几块钱一本没什么区别，但这种物理媒介在捕捉灵感、绘制草图时的灵活，是任何屏幕都无法比拟的。

我之前的塔罗牌网站，其最初的雏形就[诞生在这样一张随手涂鸦的草图上](https://liangmouyin.com/blog/13-years-tarot-website/)。

![](https://img.liangmouyin.com/2026/05/f2723dcef3d969fe7167b4c988cf5866.jpg)

HTML、Markdown 甚至是纸质本，他们并不存在取代关系。因为它们本来就是不同工具，解决的问题也都不相同：HTML 适合展示，Markdown 方便编辑或与 AI 交流，纸质本则随身携带，想到就提几笔。

HTML 只怕还无法流行起来，现在的声势浩大也只是一小群人的狂欢，无法蔓延到大洋彼岸的我们；Markdown 只怕也是这样，只属于我们这拨喜欢折腾笔记工具者的常用格式；但对于更多人，微信收藏、Word、思维导图、纸质本就能胜任生活中的所有场景。

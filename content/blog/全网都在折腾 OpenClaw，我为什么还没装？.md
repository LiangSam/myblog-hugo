我到现在还没安装 OpenClaw，这对于喜欢浅浅折腾 AI 的人来说并不科学，毕竟我连 Claude Code 都用上了。

部分读者或许并不了解 OpenClaw，你可以把它想象成一个赛博员工，可以帮你做些杂活，由于是开源的，所以只要提供办公室（一台电脑），工资（大模型 API），对讲机（聊天工具），就能自己干活（直接在电脑操作软件，如整理文件、梳理网络信息、管理日程）。但要它能做这些事，需要赋予非常高的自由度，所以像极了一个吭哧吭哧埋头干活的实习生。

这听起来多有趣，只要往飞书（或 Telegram、Discord、钉钉、微信）发送一条消息，比如「帮我在网上找找最近有没有人指名道姓骂我」，那么员工就在指定平台搜索相关信息，最后就向你汇报结果：「某月某日有个网友说你天天吃鸡腿饭，有没有考虑过鸡的感受？」，你就能思考要不要回击。

也就是说，你的电脑前不需要坐着一个人，也不用手机远程操作，它就能根据你的一句话自行运转。它在玩家手里还能发挥更多妙用，上述胡诌操作只是小巫见大巫，有人让它连夜开发工具、监控竞对、内容创作等等。

而你，只需要动动嘴就够了。

过去的 AI 都还停留在对话上，而 OpenClaw 已经迈步到「边说边干活」的境界，而且有记忆功能、能接入国内大模型、本地开源……于是，最开始几个自媒体在聊养虾（OpenClaw 的 logo 是龙虾），其他人看到了，也纷纷加入，最后整个 AI 圈、乃至国内社交媒体都在研究，就像一把鱼饵被撒入湖水中，泛起阵阵涟漪，其味道迅速吸引附近的鱼来啃食。

在我看别人用 OpenClaw 产出什么内容前，反倒是听说带动了就业。因为 OpenClaw 需要部署到本地，安装一堆工具，准备大模型 API，配置聊天工具等等，对很多人来说极其复杂，因此有人提供上门安装服务，收费几百到千元不等。而很多厂商直接提供云服务器，主打秒级部署。

后者尚能理解，毕竟人家就是买服务器的，可对于那些请人上门安装的人，大部分肯定对计算机知之甚少，终端（Terminal 或 PowerShell）都不知道如何打开，所以花钱一了百了。

但这种人真的用得上 OpenClaw 吗？如果是重度折腾者，即便不懂如何部署，也能靠着问 AI 一步步搭建，而想要尝鲜的人，他们安装来问天气、查百科、聊聊天，于是跟风购买 Mac mini，但这些打开手机软件就能查找，如果后期 API 欠费、插件冲突、或者 Github 仓库更新导致系统出错，他们能处理吗？只怕这都白瞎了。

诚然，很多人购买 Mac mini 是为了数据隔离，担心在主力设备安装后，一句话说错，龙虾就把你硬盘都清空了。

而且我对于那些大张旗鼓宣传 OpenClaw 持怀疑态度，因为我看很多人除了写用 OpenClaw 的相关文章或制作视频，比如《如何配置龙虾》，就没见其他内容，他们用 OpenClaw 来产出 OpenClaw。听起来很拗口，但就像那些尝试了各个笔记软件的人，结果里面写的东西就是笔记方法论。

今天是 OpenClaw，明天是 CloseClaw，后天呢？热点天天都在变，需求也在变，但产出是恒定且能迭代的，有就是有，没有就是没有。

在技术无限膨胀的时代，通用的经验反而弥足珍贵。去年学会的提示词结构，到今年已经换成更易用的 MCP、skill、Agent、Rag、SubAgent（没事，我也不太懂）……但诸多工具的核心依然是你希望用来解决什么问题，以及如何用语言把模糊笼统的目标，清晰地表达出来，并要拆解成 AI 能听懂、能执行的子任务。

我想，只要你界定好输入和输出，这么黑匣子里养的是龙虾还是螃蟹，其实并不重要。

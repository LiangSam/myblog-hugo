前阵我终于找到一个[顺手的本地音乐播放器](blog/try-macos-music-player/)，但当天我就冒出一个新思路，甚至觉得有几分意思，就是用 AI 来协助管理音乐。

须知过去管理本地音乐劳心费神，盖因下载来的音乐文件质量良莠不齐，比如某首歌只有 MP3 格式，后来好不容易找到 FLAC 无损音乐，可文件一多，也就无暇排查有哪些文件是重复的。加之文件命名混乱，有的歌曲命名为 `歌手-歌曲`，而有的是反过来，导致你没法在文件夹按命名来梳理。

而我那天整理时，鬼使神差，在音乐库文件夹打开 Claude Code，让它先看看我的音乐文件夹，有哪些是重复的，再帮我删掉即可。

果然，里面还真有几首歌是重复的，我再动动嘴让它删掉，一下子就解决了好大难题。这些文件是过去积累了几年的，近几年我主要听流媒体，疏于管理，加上觉得一团乱麻，换过几次文件，更不愿意打开。而现在变成一个干净的音乐库，我也乐意打开它听，起码不受订阅服务和网络的限制。

除查重外，还能让 AI 把文件按照你想要的方式来命名，专辑单独建文件夹，检查音乐库里有哪些歌手的歌，又有多少是 Live 版本，甚至还能批量整理元数据。

可处理文件只是第一步，AI 也不该止步于此。因为文件名就是最好的数据，你甚至不用和 AI 说你有哪些音乐，而是在终端打开 Claude Code 等工具（或者提取音乐库的文件树，你就得到一份音乐列表，直接交给 AI 对话），我们就能发掘有意思的东西。

**首先是人格侧写。**

之前我看不少自媒体把日记和文章喂给 AI 来做分析，发现文字之海中自己未曾发现过的东西，但我只试过把文章喂给 NotebookLM 来分析我的写作风格，而现在早已忘得一干二净；而在这个思路上，我们也能对音乐来做分析，你听什么歌代表了你的喜好和审美。

Claude 就说我的音乐有不少重复，甚至会对同一首歌，收藏不同歌手版本。比如《传奇》就有李健和王菲版本，收藏了大量的 Live 版音乐，规模最大的是张国荣，甚至出现了异类万能青年旅店，以及居然有一首蔡徐坤的《Deadman》（AI 表示震惊），顺带说我收藏了一整张 *In Bruges* （杀手没有假期，2008）的 OST，这说明发生了某种强烈的关联，并专门收藏。这话没错，我是真喜欢这张专辑。

然后有意思的来了，AI 说我唯独少了几个艺人的歌，他们的地位也不低，如周杰伦、五月天、林志炫，我是一首都没下载。林志炫倒是说对了，我之前挺喜欢他的音乐，只是忘记下载而已，他在《歌手》唱的那些歌我都挺喜欢。

![Claude 的解读](https://img.liangmouyin.com/2026/05/e4d7253d482a22eb320a910fa1706f75.jpeg)

不过这也仅供一乐，不必太过当真，因为也存在过分解读。具体提示词如下，诸君自取：

```
请根据音乐文件列表，对收藏者进行人格侧写。要求：
- 推断性格，而非描述歌单
- 每个结论给出具体的文件证据
- 重点关注异常值、孤立选择、以及明显缺席的音乐类型
- 寻找跨艺人的隐藏规律（同一首歌的多个版本、艺人间的隐性关联、对某艺人的选曲偏向）
- 禁止使用「感性」「细腻」「有品味」等通用描述
```

**再之后是推荐音乐。**

有了一个列表和侧写后，AI 就知道你喜欢哪一类音乐，你就能向外探索音乐，再不局限于当前风格的同时，也能避免踩雷（我曾给不少朋友听过左小祖咒的歌，但纷纷面露难色，我想你并不打算体验）。

Claude 看我有好几首孙燕姿的歌，就推荐我听一下陈绮贞的歌，但它还有自己的理解，说《旅行的意义》最出名，但并不是她最好的歌，真正值得听的是《华丽的冒险》，从头到尾很完整。尽管我听过她的歌，这次从善如流，听罢整张专辑后，我只觉得慵懒极了，气质散漫，我表示喜欢后，Claude 又推荐台湾歌手魏如萱（真没听过），说风格相近，但更飘更跳脱，果不其然，好极了。

![Claude 的推荐音乐](https://img.liangmouyin.com/2026/05/f66878f946953494f653bd71d160f1f8.jpeg)

我想通过这个，你暂时应该不会陷入音乐荒，这比流媒体的推荐更加体系和完整，因为流媒体通常只能推荐一首歌，而且还是无疑是听过去的，好听也就止步于收藏，很少专门搜索并听完整首专辑。

```
根据我的音乐收藏列表和收藏风格，找出我可能会喜欢、但明显不在我收藏范围内的音乐。
不要推荐风格高度相似的，要推荐那些「如果你真的理解这个人，他应该会喜欢但还没发现」的作品。 
每首给出推荐理由，说明它和我已有品味的具体连接点。
推荐完之后，告诉我你认为我最可能不同意的那一首是哪首，以及为什么你还是推荐了它。
```

上述方法不是非得把音乐下载才行，你直接甩个歌单链接也可以。不过网易云音乐的链接我用 DeepSeek 测试后，发现并不行，因为查看歌单需要登录，而 DeepSeek 显然不听音乐所以没账号（它花了几分钟了读取链接，以失败而告终）。然后我试图在浏览器复制整个歌单，最后也不行。无解，网易云你赢了。

我其实还尝试过让 Claude 帮我下载音乐到本地，这样岂不更方便。不过它的道德意识极强，拒绝我的要求，我也不再折腾，听什么就到 Apple Music 搜索，如此相得益彰。

这就是比较荒诞的一幕，即便已有本地音乐，可当我想听歌时又丝滑地回到流媒体，毕竟后者可以直接听歌，免去找网站的繁琐步骤。好在我还有会员，还在我还能多听一会儿，那我还折腾个啥，至少本地音乐的价值已经增加了不少，那就该回到听歌上。

本想找 AI 写代码，它却甩给我一个软件

2025-12-11 #工具

自从用 MacBook 以来，我很喜欢它的触控板设计，起码比我几年前用的笔记本好，动作多，操作丝滑，有时你感觉按下去，其实是马达在震动。可一旦连上显示器、鼠标和键盘时，触控板自然无用武之地，只能依赖鼠标滚动。

然而 macOS 一切都是跟 Windows 相反的，窗口管理的交通灯、内侧的 Command 承担 Ctrl功能、上下任务栏，乃至鼠标的滚动也截然相反。触控板胜在符合直觉，用起来无感，可一旦接入鼠标，当我习惯性往下滑，页面却往下滚，在 Windows 上是往上面的，这很难适应。

后来才知道，需要去系统设置的触控板里，关掉自然滚动（Natural scrolling）。如此鼠标滚动方向逆转，然则关掉蓝牙，断开鼠标只用触控板，又需要重新调一次。我找到最好的方法，就是唤出 LaunchBar，输入 Tra，进入触控板设置，再手动开关。

就这样适应两三个月，其间反复切换，直到这周，我想到解决方案：即让 Gemini 帮我写一个 Keyboard Maestro 动作，当蓝牙连上我的鼠标时，就把自然滚动关闭，如果蓝牙断开，就打开。

我自认为开启 Vibe Coding，也准备好跟它反复切磋试错，谁知道 Gemini 直接甩出一个开源软件给我，说这个可能会更好，当然，如果您想用 KM，那也行，按我下面的教程来。

这个开源软件是 Mos，专门用来实现触控板和鼠标方向分离，而且还能让鼠标的滚动像触控板一样丝滑，不管蓝牙断联几次，它依然运行。很明显，这一听就比我那灵机一动的方案好几倍。

我立即下载安装，除了开机自启和隐藏状态栏图标，其他不需要额外设置。让我惊讶的是，鼠标滚动居然丝滑了起来，原来会偶有卡顿，可不得其法，只能将就，如今觉得腿脚利索了。

Mos 这样的轻便软件，也许是某些人的装机必备。但我这个门外汉一无所知，觉得有 Keyboard Maestro 这等神器足矣，遇到问题也是想着它能不能解决；岂不知，很多基础性问题，在我遇到前想必已成为共识，那么解决方案也很多样，只是如今有 AI，我猜想着该用牛刀来杀鸡。

事实上，我连鸡也不用亲自宰，去沙县就能吃上鸡腿饭。

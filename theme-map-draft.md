# 主题地图草稿

目的：为未来的 `/map/` 页面提供第一版策展底稿。

这份草稿不是 tag 云，也不是最终数据。它把文章按“长期问题”组织：母题 -> 子主题 -> 代表文章 -> 关系线。后续如果方向对味，再整理成 `data/theme_map.yaml` 供页面读取。

## 设计原则

- 母题少而稳定：控制在 8 个左右，让读者能记住。
- 子主题负责解释母题：每个母题 3-5 个子主题。
- 文章只放代表作：不要把 752 篇全部塞进地图。
- 关系线有语义：只保留能帮助理解的连接，不做毛线团。
- AI 生成底稿，作者确认发布：AI 负责覆盖和补漏，最终解释权保留给作者。

## 关系线草稿

关系类型：

- `同题`：围绕同一个问题反复写。
- `延伸`：一个问题自然发展出另一个问题。
- `修正`：后文修正或降低前文的判断。
- `转向`：从表面主题转向更深层主题。

候选关系：

- 工具不是目的 -> 读书、打印与慢一点｜延伸｜打印机从工具变成阅读秩序。
- 工具不是目的 -> AI 作为外置器官｜延伸｜AI 是更复杂、更危险的工具。
- 工具不是目的 -> 睡眠、身体与生活秩序｜转向｜工具最终要降低身体摩擦。
- 写作作为自救 -> 关系、边界与成年人｜延伸｜公开表达会改变关系现场。
- 写作作为自救 -> 深圳与漂泊｜延伸｜城市经验通过写作被保存。
- 普通人的钱与欲望 -> 关系、边界与成年人｜同题｜钱、人情、借贷都在考验边界。
- 普通人的钱与欲望 -> 睡眠、身体与生活秩序｜转向｜消费决策最终落到生活质量。
- AI 作为外置器官 -> 写作作为自救｜修正｜AI 可以辅助写作，但不能替代自我描述。
- 读书、打印与慢一点 -> 写作作为自救｜延伸｜输入方式会改变输出方式。
- 深圳与漂泊 -> 普通人的钱与欲望｜同题｜城市生活塑造存钱、租房、消费和安全感。

## 第一版页面建议

- 首页：只放主题地图预览窗，展示 3-4 个母题岛和少量文章卡。
- `/map/`：完整白板画布。
- 默认视图：母题岛 + 子主题 + 每个母题 2-3 张代表文章。
- 点击母题：聚焦该岛，弱化其他岛。
- 点击文章：右侧详情栏显示摘要、日期、相关母题、阅读入口。
- 关系线：默认少量显示；hover 或选中后显示相关关系。
- 移动端：不要强行白板缩放，改成母题折叠列表。
